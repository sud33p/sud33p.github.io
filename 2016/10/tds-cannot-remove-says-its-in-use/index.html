TDS cannot remove template and says it's in use.. but it isn't!!!

October 20, 2016 in Sitecore, Team Development for Sitecore · 2 minute read

Have you seen this error before? You normally get this on the TDS sync window when TDS says the template is Missing in Project and you will choose to Remove from Sitecore but the error window says nah, can’t do it.

An exception occurred while updating the sitecore item /sitecore/item/path/name.

The "name" template is in use.

Delete all the items that are based on this template.

Please see the output window for more details.
TDS says the template is in use

First of all, the word “occured” in the screenshot is incorrect, says Wiktionary. But nothing worse than getting this error when you are trying to do a sync on a long list of new/deleted/updated items/templates/whatever, presumably after isolating yourself on your solo “feature” branch for a while.. Actually, the worst one has to be the one where TDS does not have a clue what to blame and just throws an exception. And then you have to scroll down the sync window to find the broken items and manually remove them.

This one is equally bad because there isn’t any item that’s using the named template. You know there isn’t, but TDS says otherwise. So you have to investigate. “Links” button on the “Navigate” ribbon to the rescue.

Content Editor > Navigate (ribbon) > Links

Checking the Links referral

Guess what? No content item is using it. The Links window shows a rendering, but it shouldn’t as it was updated to use something else, I swear. Let’s check further.

Checking the rendering datasource template
Checking the branch item template

So further investigation proves that the listed rendering is not using the template at all. What now?

Solution

It is now obvious that the Links window is not reporting correctly. So definitely a problem with the Links database. Let’s go ahead an rebuild it, the master database at a minimum as that’s our concern here.

Control Panel > (Database) > Rebuild link databases

Rebuild Link Databases

Now check back on the Links window, there is nothing referring to our template now.

Checking the Links referral again

Go ahead and re-sync TDS. It should be able to remove the template now.

Conclusion

So looks like TDS goes through the same sitecore middle layer or uses it for its operations. So next time, if you get this error and you are pretty sure the template is not in use, just go ahead and try your luck by rebuilding the links database.
