Sitecore Launchpad apps made easy with Bookmarks

December 15, 2016 in Sitecore · 2 minute read

I am not sure if it’s the pure laziness in me, or an interest in learning new techniques, I try to find easier or quicker ways to do somewhat tedious things (where I can). It must be the laziness, coz I know at times I have wasted my time on some simple tasks, trying to improvise the process, which would have been done quicker the normal way. Like when trying to extract some texts from a 10 line document using RegEx or the very handy sublime text tricks, where it would have taken fraction of that time by simply deleting the unwanted texts manually.

But other times I have come up with or discovered some nice little ways to save time in otherwise the routined workflows. Not really groundbreaking stuffs, but makes a lot of difference in the long run when you are doing the same or similar stuffs repeatedly again and again. In this instance, I am talking about saving some clicks, and page loads.

If you have worked with Sitecore for a while, in the version 6 or earlier days, you would remember that we didn’t have the luxury of Launchpad back then. You would spend most of the time in the desktop mode because that’s how you access most of the apps, from the start menu. Or the desktop shortcuts, which I almost never used, and I doubt most people even knew of. The biggest problem with the desktop mode (apart from being a memory hog) was, and it still is, that you would lose the state of the opened windows if you refresh the browser. And as developers, that’s what you do most of the time.

[Image of desktop mode]

Fast forward to 7.1 / 7.2, but really the version 8 and we have now a very stable and handy Launchpad. And as the name suggests, it is the launch pad for the commonly used sitecore applications, and also the new modern SPEAK apps. So you don’t have to rely much on the desktop mode now.

[Image of launchpad]

However in saying that, now you need to come back to the Launchpad whenever you need to switch to a different app. This is still far from a painful experience, but I would definitely prefer to cut the “middle” page out. And so the hunt began.
